La mayoría de usuarios solo necesita una parte pequeña de Jekyll para mantener un sitio hijo.

| Concepto | Uso en unaltraweb |
|---|---|
| Front matter | Seleccionar layout, idioma, perfiles y metadatos de navegación |
| Colecciones | Guardar capítulos, resultados, tesis y documentación |
| Includes | Reutilizar fragmentos HTML/Liquid del núcleo |
| Layouts | Definir familias de página como perfiles, manuales y docs |
| Ficheros de datos | Guardar listas estructuradas bajo `_data` |

Ante la duda, edita contenido y front matter en el sitio hijo. Edita el núcleo solo si el comportamiento debe compartirse entre muchos sitios.
